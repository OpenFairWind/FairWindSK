\section{El área de aplicación}

El Área de Aplicación es el principal espacio de trabajo entre las barras.
Aquí es donde FairWindSK muestra:

\begin{itemize}
  \item launcher pages
  \item chart and navigation apps
  \item Settings
  \item My~Data
  \item embedded Signal~K web applications
\end{itemize}

La vista activa no debe superponer visualmente el Top Bar o Bottom Bar a menos que un
El cajón formalmente definido está cubriéndolo temporalmente.

\section{Modelo de una sola ventana}

FairWindSK impone un diseño de una sola ventana en todas las plataformas.
Ninguna aplicación abre una segunda ventana; no se lanza ninguna aplicación nativa externa.
\code{file://} launcher entries are blocked.

Android es la excepción de plataforma intencional para aplicaciones nativas seleccionadas.
En Android 13/API 33 o más reciente, FairWindSK puede actuar opcionalmente como la aplicación Home y
launch activities explicitly selected in \ui{Settings~> Android}. Those activities
Deje temporalmente la ventana FairWindSK y vuelva a través de Android Back o Home;
todo el contenido de FairWindSK y Signal~K sigue utilizando la concha de una sola ventana.

En plataformas de escritorio (Linux, macOS, Windows, Raspberry~Pi~OS),
press \key{SHIFT+TAB} to bring FairWindSK back to the foreground if a hosted web
la aplicación se ha centrado plenamente.

\section{Valores de navegación lectura correcta}

\subsection{COG and SOG}

El COG es el curso sobre el suelo; el SOG es la velocidad sobre el suelo.
Estos muestran cómo el barco realmente se mueve sobre la tierra, que puede diferir de
donde el arco está apuntando debido a la leeway y la corriente de marea.

\subsection{Heading}

La dirección te dice dónde se apunta el arco.
Si el rumbo se vuelve estancado o desaparecido, dirigir conservadoramente y cruzar con un
Compass de confianza.

\subsection{Valores de ruta y punto de referencia}

Cuando el enrutamiento es activo, BTW, DTG, TTG, ETA, XTE y VMG se actualizan continuamente.
Si alguno de estos valores deja de actualizar, compruebe el gráfico y sus alrededores antes
confiando en la guía de ruta.

\section{Configuración de la barra superior}

Open \ui{Settings~> Top Bar}.
Una vista previa en vivo de la barra aparece en la parte superior; la paleta widget está debajo.
Use the \iconlg{arrow-left-google}~move-left and \iconlg{arrow-right-google}~move-right
botones para widgets de reposición.
Toggle \key{Show Icon}, \key{Show Text}, \key{Show Units}, and \key{Show Trend}
por widget.
Tap \iconlg{refresh-google}~\key{Reset Defaults} to restore the built-in layout.

\begin{tipbox}
En condiciones ásperas, una barra superior inclinada con widgets de teclas 3--4 (posición, SOG, DPT, BTW)
es más seguro que doce artículos.
Añade widgets sólo si añaden una verdadera conciencia situacional en el pasaje actual
fase.
\end{tipbox}

\section{Bares de superposición operacional}

Tapping \iconlg{alarm-pob}~POB, \iconlg{command-autopilot}~Autopilot,
\iconlg{anchor-iec}~Anchor, or \iconlg{alarm-general}~Alarms replaces the normal Bottom
Iconos de barra con una barra operativa dedicada.
La zona de aplicación continúa corriendo detrás de la barra.
Tap \iconlg{close-google}~close (rightmost icon) to return to the normal Bottom Bar
sin cambiar ningún estado operativo.

\begin{warningbox}
Las barras de operaciones no son modales.
Los datos siguen actualizando.
El plotter gráfico y todas las aplicaciones permanecen en directo detrás de la barra operativa.
\end{warningbox}

\chapter{El lanzador y la gestión de aplicaciones}
\label{ch:launcher}

El lanzador es la posición segura de la cáscara.
Úsalo para volver a los azulejos de aplicaciones, cambiar entre tareas, dejar una aplicación congelada o
reorientarse después de un flujo de trabajo confuso.
Cada vez que la pantalla actual deja de sentirse clara o confiable, toque
\iconlg{applications}~\key{Apps} to return here.

\screenshot{launcher-overview}{Pantalla de inicio de lanzamiento: cuadrícula de azulejos, indicador de página, Bottom Bar}{}

\section{Apertura de una solicitud}

\begin{enumerate}
  \item Tap \iconlabel{applications}{Apps} in the Bottom Bar.
  \item Navigate pages by swiping if needed.
  \item Tap the tile for the application you want.
  \item Wait for it to load in the Application Area.
\end{enumerate}

Si una aplicación no aparece correctamente, evite pulsaciones rápidas repetidas.
Espere brevemente, vuelva a entrar una vez si el shell ofrece la opción, luego vuelva al lanzador
si la aplicación todavía se ve mal.

\section{Interruptor entre aplicaciones abiertas}

Utilice la tira de aplicaciones abiertas en la barra de base para cambiar entre aplicaciones cargadas
con un solo toque.
La tira muestra un pequeño icono para cada aplicación cargada; la activa es
resaltado.

\section{Añadiendo una aplicación personalizada}

\begin{enumerate}
  \item Open \ui{Settings~> Applications}.
  \item Tap \iconlabel{define-new-application-in-palette}{Create app}.
  \item Enter a display name and the full URL (\code{http://} or \code{https://}).
  \item Optionally set a custom icon path and zoom level.
  \item Tap \key{Save}.
\end{enumerate}

\section{Recuperar una aplicación Stuck o Blank}

\begin{enumerate}
  \item Wait briefly. Retry once if the shell offers a retry action.
  \item Tap \iconlabel{applications}{Apps} to return to the launcher.
  \item Check shell health and connection state.
  \item Reopen the app only after confirming the shell itself is healthy.
\end{enumerate}

\begin{tipbox}
Una aplicación fallida no significa que FairWindSK haya fallado.
La cáscara, los widgets de datos y las barras operativas (POB, ancla, piloto automático) continúan
trabajar independientemente de cualquier aplicación individual.
Una aplicación degradada no es la misma que un estado de buques degradados.
\end{tipbox}
